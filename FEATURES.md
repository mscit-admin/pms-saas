# مميزات النظام — PMS (مراقبة وإدارة تذاكر Jira)

نظام لمراقبة وإدارة تذاكر Jira: مزامنة، لوحات مؤشّرات، تحليلات إدارية، إدارة
الاعتماديات، صلاحيات دقيقة، وأمان. هذا الملف مرجع موثّق لما تم إنجازه.

> الفرع: `claude/gifted-archimedes-el3240` — آخر تحديث: 2026-06-21

---

## 🔗 التكامل مع Jira والمزامنة
- مزامنة كاملة عبر **JQL** (Polling) مع ترقيم تلقائي للصفحات.
- **Webhook** للتحديث الفوري عند تغيّر التذاكر.
- زر **«مزامنة جيرا»** داخل التطبيق (صلاحية `trigger_sync`) مع **سجلّ مزامنة**.
- جلب: الحالات، المسؤولين، الأولويات، الوسوم (Labels)، نوع العمل، وسجلّ التغييرات (history).
- **لقطات يومية** للاستثناءات وتوزيع WIP لحساب الاتجاه والتدفّق.

## 📊 لوحة المعلومات (Dashboard)
- شاشة واحدة موحّدة، عناصرها قابلة **للسحب والإفلات** وإعادة الترتيب في عمودين، وقابلة **للطيّ**.
- **مؤشّرات KPI** (كل مؤشّر بصلاحية مستقلة):
  - إجمالي التذاكر · المفتوحة · المنجَزة · المتأخرة · بلا مسؤول · الراكدة · مراجعة متأخرة · خرق SLA · متوسط زمن الدورة.
- **عناصر (Widgets):**
  - أعباء الفريق · WIP عبر الزمن · الإنتاجية والتنبؤ · اتجاه الاستثناءات · زمن الدورة حسب الأولوية · الزمن في كل مرحلة.

## ⚙️ التشغيلي (Operational)
- شاشة **الاستثناءات**: تذاكر مخالفة للقواعد مع فلاتر (مسؤول/مشروع/أولوية/حالة/وسوم/نوع) وإخفاء المؤجَّل/المُقَر.
- شاشة **كل التذاكر المفتوحة** مع نوع العمل والترقيم.
- متابعة الاستثناءات: **إقرار / تأجيل / تعيين مالك** (صلاحية `manage_exceptions`).
- **إجراءات على التذاكر** من داخل التطبيق: إسناد · نقل الحالة · تعديل الحقول · تعليق (صلاحية `act_tickets`).
- **تعليق ذكي** مقترَح بالذكاء الاصطناعي + **إشارات @** بأسماء مستخدمي Jira.

## 📈 الإداري / التحليلات (Managerial)
- **تقييم الأداء** للفِرق.
- **صحة المشاريع (RAG)**.
- **تدفّق العمل والاختناقات**: WIP حسب المرحلة · اختناقات المشاريع · أقدم العناصر العالقة (**ترقيم 5 لكل صفحة**).
- **تنبؤ SLA** + ضبط قواعد SLA (صلاحية `manage_sla`).
- رسوم بيانية SVG أصلية (خطوط/مساحات/أعمدة) متجاوبة على الجوال.

## 🔀 الاعتماديات (Dependencies)
- شاشة **«اختناقات الاعتمادية»** المخصّصة (تذاكر حاجبة).
- **إنشاء** اعتمادية من التطبيق (تعتمد على / يعتمد عليها).
- **إلغاء يدوي** للاعتمادية مع **سبب إلزامي** (تحقّق في الواجهة والخادم).
- **إلغاء تلقائي** عند بلوغ التذكرة الحاجبة حالة إلغاء **قابلة للضبط من الأدمن** (مع مفتاح تفعيل، لأن حذف روابط Jira تغيير فعلي).
- **سجلّ الاعتماديات المُلغاة** (من / متى / لماذا · يدوي/تلقائي) بصلاحية `view_dependency_log`.
- زر **«تراجع»** يعيد إنشاء الرابط المحذوف في Jira.

## 🔔 التنبيهات
- اكتشاف الاستثناءات الجديدة وإرسال تنبيهات عند تهيئة قنوات الإشعار.

## 👥 المستخدمون والأمان
- إدارة المستخدمين.
- **أدوار وصلاحيات** دقيقة (كتالوج ~30 صلاحية).
- **تحقّق ثنائي (2FA / TOTP)** مع إعادة ضبط (صلاحية `reset_2fa`).
- **الملف الشخصي**: تعديل الاسم وصورة شخصية مع **قصّ وتحجيم** قبل الرفع.
- **سجلّ الدخول والتدقيق** (صلاحية `view_audit`).

## 🎨 الواجهة والهوية
- تصميم **شريط جانبي بنمط ERPNext** قابل للطيّ، بأيقونات خطّية أحادية، وفئات قابلة للطيّ.
- **بحث شامل (Command Palette)** عبر `Ctrl/Cmd+K` مع سجلّ.
- دعم **العربية/الإنجليزية** و**RTL**، وتجاوب كامل على الجوال.
- **الهوية** (صلاحية `manage_branding`): شعار / خلفية / أيقونة.

## 🧠 الذكاء الاصطناعي
- وحدة ذكاء قابلة للضبط من الإدارة (مفاتيح وإعدادات) لاقتراح التعليقات.

## 🗂️ كتالوج الصلاحيات (مرجع)
| المفتاح | الوصف |
|--------|-------|
| `view_dashboard` | عرض لوحة المعلومات |
| `view_operational` | عرض التبويب التشغيلي |
| `view_managerial` | عرض التبويب الإداري |
| `kpi_*` | إظهار/إخفاء كل مؤشّر KPI على حدة |
| `widget_*` | إظهار/إخفاء كل عنصر لوحة على حدة |
| `manage_sla` | ضبط SLA |
| `trigger_sync` | تشغيل المزامنة |
| `act_tickets` | إجراءات على التذاكر (إسناد/تعليق/نقل/ربط) |
| `manage_exceptions` | متابعة الاستثناءات (إقرار/تأجيل/مالك) |
| `manage_users` | إدارة المستخدمين |
| `manage_roles` | إدارة الأدوار والصلاحيات |
| `manage_settings` | إدارة الإعدادات |
| `manage_branding` | إدارة الهوية |
| `manage_integration` | إدارة الربط بجيرا |
| `view_audit` | عرض سجلّات الدخول والتدقيق |
| `view_dependency_log` | عرض سجلّ الاعتماديات المُلغاة |
| `reset_2fa` | إعادة ضبط التحقق الثنائي |

## 🚀 النشر
```bash
git pull origin claude/gifted-archimedes-el3240
npm run db:migrate    # ترحيل المخططات (جداول/أعمدة جديدة)
npm run seed:admin    # مزامنة كتالوج الصلاحيات لدور المسؤول
npm run build
sudo systemctl restart jira-monitor-web
```

> بعض المزايا تتطلّب منح صلاحياتها الجديدة من **الإدارة ← الأدوار والصلاحيات**
> (مثل `view_dependency_log`)، وتفعيل خياراتها من **الإعدادات** (مثل حالات إلغاء
> الاعتمادية والحذف التلقائي).
